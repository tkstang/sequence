## Structural Conventions

- Component tests: `*.test.tsx` co-located in `src/`, run by vitest under jsdom with `vitest.setup.ts` (RTL matchers + cleanup); automatic JSX runtime means no `import React`.
- Non-component logic tests: `*.test.ts` in `src/` (vitest, no DOM).
- E2E: `*.spec.ts` under `apps/web/e2e/` only (Playwright `testDir: ./e2e`; vitest `include` excludes `e2e/`).
- Relative imports keep explicit `.ts`/`.tsx` extensions; the `@/` alias maps to `src/` (mirrors `tsconfig.json`).
- Leaf game UI components are prop-driven from `GameSnapshotView` (stay usable from tests/fixtures).

## Behavioral Conventions

- Import `AppRouter` from `@sequence/api` as a **type** only; ship no API runtime code in the browser bundle.
- The WS client is lazy so auth pages don't open unauthenticated sockets before a session cookie exists.

## Counter-Examples

- Writing an e2e test as `src/foo.test.ts` → it runs under vitest/jsdom (no browser) and fails; it must be `e2e/foo.spec.ts`.
- A component test without the jsdom env/setup → RTL matchers/`document` unavailable.
- Importing API runtime values (not just types) from `@sequence/api` → leaks server code into the browser bundle.
- Adding `import React` in a component → unnecessary (automatic JSX runtime).

## New-File Workflow

1. New component → `Component.tsx` + co-located `Component.test.tsx` (vitest+jsdom).
2. New end-to-end flow → `e2e/<flow>.spec.ts` (Playwright); run with `pnpm --filter @sequence/web e2e` when `DATABASE_URL_TEST` is present.
3. Keep leaf components prop-driven.

## Preferred Default for New Files

- Component test → `*.test.tsx`; e2e test → `e2e/*.spec.ts`; pure logic test → `*.test.ts`.
