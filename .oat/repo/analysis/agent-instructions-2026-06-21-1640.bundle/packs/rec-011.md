## Evidence

- `packages/api/src/test/harness.ts` — `createHarness()` + `DATABASE_URL_TEST` + truncate-per-test.

## Structural Conventions

- A glob rule fires at file-creation time on test globs and states which runner/setup applies based on suffix + location.
- If created across both providers, keep the rule **body byte-identical**; vary only frontmatter (`paths:` for claude, `globs:` for cursor).

## Behavioral Conventions

- E2E → `apps/web/e2e/**/*.spec.ts` (Playwright; needs `DATABASE_URL_TEST`).
- Web component → `apps/web/src/**/*.test.tsx` (vitest jsdom + `vitest.setup.ts`).
- Web/logic unit → `**/*.test.ts` (vitest node).
- API integration → `packages/api/src/**/*.test.ts` via `createHarness()` (single-fork, truncate-per-test).

## Counter-Examples

- One umbrella rule that blurs the four setups (the file-type checklist prefers splitting when setups differ materially).
- Diverging rule bodies between `.claude/rules` and `.cursor/rules`.
