## Findings

### Critical

- **Game route does not expose save-and-exit or concede controls** (`apps/web/src/app/game/[id]/page.tsx:264`)
  - Issue: p06-t12 requires "concede + save & exit reachable on both" layouts, and FR10/FR11 are P0 lifecycle flows. The rendered game route includes board, sequence choice, handoff, drag tray, hand, toasts, and rematch after finish, but there are no `saveAndExit` or `concede` mutations or buttons in the game UI. A user in an active web game cannot deliberately save or concede from the implemented route.
  - Fix: Add accessible lifecycle controls on the active game screen for `game.saveAndExit` and `game.concede`, with confirmation/error toasts and responsive placement at 375px and desktop. Cover both controls with component tests and at least one browser path.
  - Requirement: FR10, FR11, p06-t12

- **Playwright suite does not cover the required p06 critical paths** (`apps/web/e2e/game-route.spec.ts:9`)
  - Issue: p06-t13 explicitly called for browser coverage of join->play->win, reconnect recovery, rematch, and pass-and-play. The committed suite has one test that creates a local game, makes one tap-mode move, checks viewport bounds, and checks the handoff veil in desktop and mobile projects. It does not exercise two-context join/play/win, reconnect, rematch, or hard-mode drag/full-game behavior required by FR6/FR7/FR12/FR16 verification.
  - Fix: Add the missing Playwright scenarios from p06-t13: remote join->play->win, reconnect/reload recovery, rematch routing/roster retention, and pass-and-play handoff. Include hard-mode drag coverage or an equivalent browser-level verification for FR7's mobile/desktop acceptance.
  - Requirement: FR6, FR7, FR12, FR16, NFR3, p06-t13

### Important

- **6-player randomize conflicts with the approved web lobby shape** (`packages/api/src/game/routes/randomize-teams.ts:20`)
  - Issue: The p06 web lobby treats 6-player games as 3 teams of 2 (`LobbyTeams.tsx:54`) and enables start only when each of those three teams has two players (`LobbyTeams.tsx:77`). API randomize currently hard-codes 6-player games to 2 teams (`TEAM_COUNT[6] = 2`), so pressing Randomize in a full 6-player web lobby can assign teams `1/2/1/2/1/2`, leaving Red empty and the web Start button disabled. That breaks a creator control required by FR4/p06-t02.
  - Fix: For p06, make randomize follow the web-selected/approved 6-player shape (3 teams of 2), or add an explicit team-count setting that the web UI and API both honor before randomizing and start-gating. Add an integration/component test proving a randomized 6-player web lobby remains startable.
  - Requirement: FR4, p06-t02

- **Recent-event replay can leave clients with a stale move version** (`packages/api/src/game/routes/on-game-event.ts:91`)
  - Issue: Snapshot recovery carries the current `version`, and live broadcasts carry post-commit `version`, but the recent `lastEventId` replay path selects only `seq`, `type`, and `payload` from `game_events`. The reducer preserves the old version when an event has no version (`game-state.ts:188`), and the page's mutation error handler only shows a toast on `CONFLICT` rather than forcing the p06-t01 "refetch via resubscribe" recovery. A client that replays missed events rather than receiving a snapshot can render the latest board while still submitting an old version.
  - Fix: Persist or reconstruct post-event versions for replayed events, or send a current snapshot/version after gap replay. Also implement the p06-t01 `CONFLICT` recovery path in the page so stale-version failures resubscribe/refetch rather than requiring a manual reload. Add a test where a replayed missed move advances the client version before the next mutation.
  - Requirement: FR6, p06-t01

### Medium

- **Concede result screen lacks attribution** (`apps/web/src/app/game/[id]/components/game-state.ts:427`)
  - Issue: p06-t10 calls for concede attribution on the game-over screen. The reducer drops the `GameConceded.payload.team` and only stores `endReason: 'concede'`; `GameOver` therefore renders only "Game conceded" with no conceding team/player context.
  - Fix: Preserve the conceded team or seat in `GameViewState` from `GameConceded`, pass it to `GameOver`, and render a short attribution such as "Team 2 conceded" or the conceding player's/team's name.
  - Requirement: p06-t10, FR11
