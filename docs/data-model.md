# Data Model

Sequence Online persists to Postgres (Neon) through the Drizzle ORM. The schema
is defined in TypeScript under `packages/api/src/db/schema`; generated SQL
migrations live under `packages/api/drizzle`. For the generate/migrate workflow
see [development.md](development.md); for how this data flows through the runtime
see [architecture.md](architecture.md).

The schema barrel `packages/api/src/db/schema/index.ts` re-exports every table so
the Drizzle client and `drizzle-kit` see one canonical schema object. The
drizzle-kit config (`packages/api/drizzle.config.ts`) points `schema` at that
barrel and `out` at `./drizzle`, dialect `postgresql`.

Three game tables are API-owned: `games`, `game_players`, `game_events`. Four
more (`user`, `session`, `account`, `verification`) are owned by Better Auth.

## games

One row per game — the single-writer row. Board, deck, played pile, and
completed sequences live in jsonb because the game is always loaded and saved
whole by one writer in a transaction, so normalizing the 100 board cells would
only add joins.

| Column | Type | Notes |
| --- | --- | --- |
| `id` | uuid | Primary key, `gen_random_uuid()` default. |
| `invite_code` | text | Not null. Unique index `games_invite_code_idx`. |
| `created_by` | text | Not null. Holds Better Auth `user.id` at the application layer — no DB-level FK (the `user` table is owned/generated by Better Auth). |
| `rematch_of` | uuid | Self-FK to `games.id`, `ON DELETE SET NULL`. Points at the game this rematch was spawned from. |
| `local` | boolean | Not null, default `false`. Local (single-device) game. |
| `player_count` | smallint | Not null. Immutable after start. |
| `mode` | text | Not null. `GameMode` = `'tap' \| 'drag'`. Immutable after start. |
| `timer_seconds` | integer | Nullable. Per-turn timer length, or null for untimed. Immutable after start. |
| `status` | text | Not null, default `'lobby'`. `GameStatus` enum (see below). |
| `version` | integer | Not null, default `0`. Optimistic-concurrency counter (see [Optimistic concurrency](#optimistic-concurrency)). |
| `current_seat` | smallint | Nullable. Seat whose turn it is. |
| `round` | integer | Not null, default `0`. |
| `next_sequence_id` | integer | Not null, default `1`. Monotonic id for the next completed sequence; mirrors `GameState`. |
| `turn_deadline_at` | timestamptz | Nullable. Wall-clock deadline for the current turn. |
| `turn_remaining_ms` | integer | Nullable. Milliseconds left on the current turn (used when frozen/saved). |
| `pending_choice` | jsonb | Nullable `PendingChoiceJson`. A >5-run lock selection awaiting the placer's choice. |
| `board` | jsonb | Nullable `BoardJson`. The board as a sparse position-keyed object. |
| `deck` | jsonb | Nullable `CardJson[]`. The draw deck. |
| `played` | jsonb | Nullable `CardJson[]`. The played/discard pile. |
| `sequences` | jsonb | Nullable `SequenceJson[]`. Completed sequences. |
| `winner_team` | smallint | Nullable. Winning team id. |
| `end_reason` | text | Nullable. `EndReason` = `'win' \| 'concede' \| 'expired'`. |
| `finished_at` | timestamptz | Nullable. When the game finished. |
| `expires_at` | timestamptz | Nullable. Lifecycle expiry target for the cleanup sweep. |
| `created_at` | timestamptz | Not null, `now()` default. |
| `updated_at` | timestamptz | Not null, `now()` default. |

Enums (string unions, stored as `text`):

```ts
gameStatuses = ['lobby', 'active', 'frozen', 'saved', 'finished']
gameModes    = ['tap', 'drag']
endReasons   = ['win', 'concede', 'expired']
```

Indexes and constraints:

- `games_invite_code_idx` — unique on `invite_code`.
- `games_status_finished_at_idx` — on `(status, finished_at)`.
- `games_rematch_of_games_id_fk` — self-FK `rematch_of → id`, `ON DELETE SET NULL`.

jsonb shapes (defined in `games.ts`):

```ts
// One board cell; absent keys mean empty.
interface BoardCell { chip?: number; lockedBy?: number }
// The board as a sparse object keyed by the game-logic position code (e.g. '1AC').
type BoardJson = Record<string, BoardCell>;

interface CardJson { rank: string; suit: string }

interface SequenceJson { id: number; team: number; cells: string[] }

interface PendingChoiceJson {
  seat: number;
  team: number;
  placed: string;
  cells: string[];
  additionalRuns?: string[][];
}
```

`BoardJson` round-trips the in-memory `Map<Position, BoardCell>` exactly: empty
cells are simply absent keys. The canonical (de)serialization lives in
`packages/api/src/game/state-mapping.ts`.

## game_players

One row per seat. Hands live here because the privacy boundary matches the table
boundary: a player's hand never leaves its own row except through the redacted
subscription stream.

| Column | Type | Notes |
| --- | --- | --- |
| `id` | uuid | Primary key, `gen_random_uuid()` default. |
| `game_id` | uuid | Not null. FK to `games.id`, `ON DELETE CASCADE`. |
| `seat` | smallint | Not null. Seat index within the game. |
| `team` | smallint | Not null. Team id for the seat. |
| `user_id` | text | Nullable. Better Auth `user.id` for a registered seat (application-layer reference, no DB-level FK). |
| `guest_name` | text | Nullable. Display name for a guest seat. |
| `guest_token_hash` | text | Nullable. Hash of the guest's game-scoped token. |
| `hand` | jsonb | Nullable `CardJson[]`. The seat's private hand. |
| `connected` | boolean | Not null, default `false`. Presence. |
| `last_seen_at` | timestamptz | Nullable. Presence timestamp. |
| `is_creator` | boolean | Not null, default `false`. Marks the seat that created the game. |

Indexes and constraints:

- `game_players_game_seat_idx` — unique on `(game_id, seat)` (one row per seat).
- `game_players_user_game_idx` — on `(user_id, game_id)`.
- FK `game_id → games.id`, `ON DELETE CASCADE`.

A seat is either a registered user (`user_id`) or a guest (`guest_name` +
`guest_token_hash`). A local-game seat 2 is a `guest_name` with no `user_id` and
no token — the creator's session covers both seats. See
[Guest vs user seats](#guest-vs-user-seats).

## game_events

The append-only event log. It serves triple duty: the move/audit log, the
subscription resume cursor (`seq`), and the replay source.

| Column | Type | Notes |
| --- | --- | --- |
| `id` | uuid | Primary key, `gen_random_uuid()` default. |
| `game_id` | uuid | Not null. FK to `games.id`, `ON DELETE CASCADE`. |
| `seq` | bigint | Not null. Per-game monotonic counter (Drizzle `mode: 'number'`). |
| `type` | text | Not null. Event discriminator. |
| `payload` | jsonb | Not null. The full event body. |
| `actor_seat` | smallint | Nullable. Seat that produced the event (`event.seat`). |
| `created_at` | timestamptz | Not null, `now()` default. |

Indexes and constraints:

- `game_events_game_seq_idx` — unique on `(game_id, seq)`.
- FK `game_id → games.id`, `ON DELETE CASCADE`.

Events store full truth server-side (including draws); the subscription layer
redacts per recipient at broadcast, so there is one source of truth and privacy
is enforced at the edge. See [The event log](#the-event-log).

## Cross-cutting concepts

### Optimistic concurrency

`games.version` is an `integer` (default `0`) that gives atomic whole-game
updates. Every state write is guarded on the version the caller read:
`persistGameState` issues an `UPDATE … WHERE id = ? AND version = ?` that sets
`version = prevVersion + 1`. When a concurrent writer already bumped the version,
the predicate matches zero rows and the helper throws `VersionConflictError`:

```ts
// packages/api/src/game/state-mapping.ts
export class VersionConflictError extends Error {
  constructor(readonly expected: number, readonly gameId: string) { /* … */ }
}
```

The version column is declared at `packages/api/src/db/schema/games.ts:100`. The
guard predicate and conflict throw are at
`packages/api/src/game/state-mapping.ts:230-235`. The same guard backs
lifecycle transitions (`persistLifecycleTransition`) so a concede / save /
freeze / resume can never silently clobber — or be clobbered by — a racing move.
Mutations that surface this conflict are documented in
[api-reference.md](api-reference.md).

### The event log

`game_events` is the durable record of everything that happened in a game. Its
`seq` column is a per-game monotonic counter assigned inside the move
transaction. `appendEvents` reads the current `max(seq)` for the game under the
transaction and inserts the new events sequentially; the combined
`persistGameStateAndAppendEvents` does the version-guarded state write and the
event append in one SQL statement so the move hot path is one load round trip
plus one atomic write round trip.

The `seq` value is the subscription resume cursor: the tRPC subscription
transport resends the last tracked event id (`lastEventId = seq`) on reconnect,
and the server replays from that cursor or sends a fresh snapshot. The `seq`
column is declared at `packages/api/src/db/schema/game-events.ts:30`; the
monotonic assignment is in `appendEvents`
(`packages/api/src/game/state-mapping.ts:479-512`). See the reconnect/replay
behavior in [architecture.md](architecture.md).

### The hand-privacy boundary

A seat's `hand` lives in its own `game_players` row and is never sent to other
players. Events carry full truth server-side, but the subscription layer redacts
per recipient at broadcast, and the API response mapping strips other players'
hands. The client never receives other remote players' hands. See the redaction
note in [architecture.md](architecture.md).

### Guest vs user seats

Each seat is either a registered user or a guest:

- User seat: `user_id` holds the Better Auth `user.id`.
- Guest seat: `guest_name` plus `guest_token_hash` (the hash of the guest's
  game-scoped token).
- Local second seat: a `guest_name` only — no `user_id`, no token — because the
  creator's session covers both seats.

`created_by` (on `games`) and `user_id` (on `game_players`) reference Better
Auth's `user.id` at the application layer only; there is no cross-migration
DB-level FK to the Better Auth `user` table.

### FK cascade and SET NULL behavior

- `game_players.game_id → games.id`: `ON DELETE CASCADE`. Deleting a game
  removes its seats.
- `game_events.game_id → games.id`: `ON DELETE CASCADE`. Deleting a game removes
  its event log.
- `games.rematch_of → games.id`: `ON DELETE SET NULL`. Deleting a referenced
  source game (e.g. via the expiry sweep) nulls the pointer rather than leaving
  it dangling or blocking the delete.

### The state-mapping round-trip

`packages/api/src/game/state-mapping.ts` is the single bridge between the
immutable in-memory `GameState` (owned by `@sequence/game-logic`) and its
persisted relational form:

- `loadGameState(tx, gameId)` (and the joined `loadGameStateWithPlayers`)
  assemble a `GameState` from the `games` row plus all `game_players.hand` rows,
  deserializing the jsonb columns back into the in-memory shapes (board `Map`,
  card arrays, sequences, pending choice).
- `persistGameState(tx, gameId, state, prevVersion)` writes the changed `games`
  columns and each seat's hand back under the version guard, returning the new
  version.
- `appendEvents(tx, gameId, events)` appends events with a monotonic per-game
  `seq`.

All three take a Drizzle transaction (`Tx`) so the move engine can run
load → reduce → persist → append as one atomic unit.

## Better Auth tables

`packages/api/src/db/schema/auth.ts` defines the four tables managed by Better
Auth: `user`, `session`, `account`, and `verification`. They are generated by the
Better Auth CLI and consumed (not authored) by the game code; `created_by` and
`user_id` reference `user.id` only at the application layer. `session` and
`account` carry FKs to `user.id` with `ON DELETE CASCADE`.
